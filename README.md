وظایف اصلی شرح زیر است:

۱. مطالعه الزامات (Requirements Study)
شرح: شما باید دقیقاً بفهمید پروژه قرار است چه کاری انجام دهد، چه ویژگی‌هایی داشته باشد، و کاربران آن چه انتظاراتی از آن خواهند داشت.

۲. طراحی ERD (Entity-Relationship Diagram)
شرح: ERD یک نمودار بصری است که نشان می‌دهد داده‌های پروژه شما (مانند کاربران، مقالات، دسته‌ها) چگونه به یکدیگر مرتبط هستند.

عناصر کلیدی که باید در ERD شما باشند:

کاربران (Users): افرادی که از سیستم استفاده می‌کنند.

مقالات (Articles): محتوای اصلی پروژه شما.

دسته‌ها (Categories): برای سازماندهی مقالات (مثلاً "علمی"، "خبری"، "ورزشی.")

فایل‌ها (Files): هرگونه فایل یا تصویری که به مقالات یا کاربران مرتبط است.

هدف: طراحی ساختار پایگاه داده (Database) پروژه.

۳. انتخاب و توجیه فناوری‌ها (Technology Selection)
شرح: باید ابزارهایی که برای ساخت پروژه استفاده می‌کنید را مشخص کنید.

Backend( سمت سرور) :php   
ساده و رایج برای پروژه‌های دانشجویی و وب‌محور


Frontend  (سمت کاربر): html css سریع و سبک چون بر پایه موتور V8 گوگل کار می‌کنه
✅ آسان برای یادگیری اگر HTML/CSS/JS بلدی
✅ پشتیبانی از هزاران پکیج آماده (npm)
✅ یکپارچگی با فرانت‌اند (هر دو با JavaScript)
✅ مناسب برای پروژه‌های REST API، مثل همین سایت مقالات

هدف: مشخص کردن "چگونه" می‌خواهید پروژه را بسازید.

۴. راه‌اندازی Git (Git Setup)
شرح: Git یک سیستم کنترل نسخه است که به شما کمک می‌کند تغییرات کد خود را پیگیری کرده و به صورت تیمی کار کنید (حتی اگر تنها باشید.)

وظیفه: ساخت یک مخزن (Repository) جدید در پلتفرم‌هایی مانند GitHub یا GitLab و اضافه کردن یک فایل اولیه README که توضیح مختصری درباره پروژه می‌دهد.

برای دیتابیس پروژه از ابزار قدرتمند mysql  که ارتباط اسانی با xampp دارد استفاده میشود

هدف: آماده‌سازی محیط برای مدیریت کد پروژه.

ERD کامل: نمودار ارتباط موجودیت‌ها برای چهار عنصر اصلی (کاربران، مقالات، دسته‌ها، فایل‌ها).

دسته‌بندی‌ها (Categories): لیستی از ۴ دسته‌بندی که برای مقالات خود انتخاب کرده‌اید (مثلاً: تکنولوژی، سلامتی، هنر، اقتصاد).
فناوری‌ها: لیستی از فناوری‌های انتخابی شما (شامل Node.js، CKEditor و هر چیز دیگری که ممکن است نیاز داشته باشید) به همراه توجیه مختصر برای هر کدام.

Git با README اولیه: لینکی به مخزن Git خود که فایل README.md اولیه در آن وجود داشته باشد.

 
Users → Articles:  هر کاربر می‌تونه چند مقاله داشته باشه.
Articles → Categories:  هر مقاله به یک دسته تعلق داره.
Articles → Files: هر مقاله می‌تونه چند فایل
